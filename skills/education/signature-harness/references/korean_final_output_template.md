# 한국어 최종 출력 템플릿

이 문서는 하네스가 학생에게 보여주거나 학교 제출용으로 다듬을 때 사용하는 한국어 템플릿이다. 내부 분석은 영어로 진행할 수 있어도, 학생에게 보이는 최종 결과물은 기본적으로 한국어로 작성한다.

---

## 1. 학생 확인 전 임시안 표시

학생 확인이 끝나지 않았으면 최상단에 반드시 표시한다.

```md
> 상태: 학생 확인 전 임시안  
> 아래 내용은 제공된 자료만 바탕으로 만든 가안이야. 실제와 다르거나 과장된 부분은 학생 확인 후 수정해야 해.
```

학생 확인이 끝났으면 다음처럼 표시한다.

```md
> 상태: 학생 확인 반영 완료  
> 학생이 선택·수정한 방향을 기준으로 정리한 최종안이야.
```

## 2. 학생에게 처음 물어볼 질문

```md
# 시작 질문

1. 이번 작업의 목적은 뭐야?
   - 학교 제출용 SIGNATURE 작성
   - 2학년 생기부 방향 설계
   - 진로/탐구 주제 추천
   - 수행평가 로드맵 만들기
   - 자기분석용 정리

2. 현재 관심 분야나 희망 진로가 있다면 말해줘. 아직 없으면 “아직 모름”도 괜찮아.

3. 생기부나 자료에서 이름, 주민번호, 주소, 전화번호, 가족/건강 정보는 지웠어?

4. 고1 생기부에서 마음에 드는 활동 2개와 별로인 활동 2개를 골라줘.

5. 2학년 때 실제로 해보고 싶은 프로젝트가 있어?

6. 피하고 싶은 방향은 뭐야?
   - 너무 AI 쪽으로만 가는 것
   - 너무 의학 쪽으로만 가는 것
   - 너무 추상적인 사회문제
   - 너무 어려운 실험
   - 발표 중심 활동
   - 코딩 중심 활동
   - 팀플 중심 활동
   - 기타

7. 결과물은 어떤 형태가 가장 현실적이야?
   - 보고서
   - 인포그래픽
   - 설문 분석
   - 실험/비교표
   - 발표 자료
   - 캠페인 기획안
   - 코드/프로토타입
   - 디자인/영상/카드뉴스
   - 아직 모르겠음
```

## 후보 1. [후보명]

- 위험: [너무 넓음/근거 약함/수행평가 연결 약함 등]

## 후보 2. [후보명]
- 한 줄 정의:
- 맞는 이유:
- 가능한 결과물:
- 위험:

## 후보 3. [후보명]
- 한 줄 정의:
- 맞는 이유:
- 가능한 결과물:

## 4. 최종 SIGNATURE 방향 템플릿

```md
# 최종 SIGNATURE 방향

## 한 줄 정의
나는 [문제/대상]을 [방법/관점]으로 탐구하며, [가치/역량]을 중심으로 성장하고자 하는 학생이다.

## 핵심 역량 3개
1. [핵심 역량 1]
   - 근거: [고1 기록 또는 학생 진술]
   - 2학년 활용: [어떤 활동으로 이어지는지]

2. [핵심 역량 2]
   - 근거:
   - 2학년 활용:

3. [핵심 역량 3]
   - 근거:
   - 2학년 활용:

## 성장 Seed 3개
1. [고1 활동에서 남은 질문/한계]
2. [더 알고 싶은 부분]
3. [2학년 교과와 연결할 수 있는 지점]

## 2학년 중심 질문
[학생의 중심축을 질문 형태로 작성]

예시:
- “교과 개념을 실제 학교 문제에 적용하면 어떤 개선안을 만들 수 있을까?”
- “자료 분석과 비교를 통해 특정 현상의 원인과 한계를 어떻게 설명할 수 있을까?”
- “사용자 관점에서 정보를 더 명확하고 안전하게 전달하려면 어떤 구조가 필요할까?”

## 최종 프로젝트
- 프로젝트명: [프로젝트명]
- 목적: [왜 하는지]
- 방법: [조사/비교/분석/설계/실험/제작 등]
- 결과물: [보고서/인포그래픽/설문 분석/프로토타입/정책 제안서 등]
- 예상 한계: [자료 부족/시간 부족/표본 수 부족/구현 범위 등]
- 실패해도 남길 수 있는 분석: [무엇을 배웠는지]
```

## 7. 최종 감사 요약 템플릿

```md
# 최종 점검 결과

## 근거가 충분한 부분
- [문장/주장]: [근거]

## 학생 확인이 필요한 부분
- [문장/주장]: [왜 확인이 필요한지]

## 약하게 표현해야 하는 부분
- 원래 표현: [표현]
- 안전한 표현: [수정안]

## 제거한 부분
- [제거한 표현]: [이유]

## 최종 verdict
- PASS: 학교 제출용으로 사용 가능
- WARN: 학생 확인 또는 일부 수정 후 사용 권장
- BLOCK: 근거/개인정보/과장 문제로 아직 사용 불가
```
